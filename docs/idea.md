# MathPath — Interactive Math Learning Platform

## Overview

MathPath is a web-based learning platform where students follow a structured pathway of math video lectures, take AI-generated quizzes after each lesson, and compete on a global leaderboard. The platform is built with **Angular** on the frontend and **Django + Django REST Framework** on the backend, using PostgreSQL as the database, containerized with Docker Compose.

---

## Problem Statement

Students learning mathematics independently often lack structure, accountability, and interactive feedback. Watching a video is passive — without a quiz or scoring system, retention is low. MathPath solves this by combining curated YouTube lectures with immediate knowledge checks and progress tracking, turning passive watching into active learning.

---

## User Journey

```
Register / Login
      ↓
Browse Lesson Pathway (ordered list of math topics)
      ↓
Open a Lesson → Read AI-generated Overview → Watch YouTube Lecture
      ↓
Take Quiz (AI-generated questions based on the lecture)
      ↓
Submit Answers → See Your Score
      ↓
View Global Leaderboard (ranking across all users)
      ↓
Move to Next Lesson → Repeat
```

## Core Features

### Authentication
- User registration and login
- JWT-based authentication (access + refresh tokens)
- Angular HTTP interceptor attaches token to every request
- Login and logout endpoints on the backend

### Lesson Pathway
- Ordered list of math topics (e.g., Algebra → Functions → Derivatives)
- Each lesson card shows: topic title, difficulty, completion status
- Lessons are locked/unlocked based on progress (optional)

### Lecture Page
- Embedded YouTube video player (from the team's own channel)
- AI-generated overview: a short summary of what the lecture covers, generated once and stored in the database
- "Start Quiz" button appears after the video section

### Quiz System
- Questions are generated by AI (OpenAI or Gemini API) based on the lecture title and description
- Multiple choice format (choices stored as JSON in the database)
- Questions generated on first request and cached — no re-generation on repeat visits
- User submits answers → score calculated server-side → result saved to database

### Leaderboard
- Global ranking of all users by total score across all completed quizzes
- Shows: rank, username, total score, number of lessons completed
- Updates in real time after each quiz submission

### User Profile
- Personal stats: lessons completed, average score, total points
- History of past quiz attempts with scores per lecture

---

## AI Integration

All AI calls are made **server-side** in Django to keep API keys secure and to cache results in the database.

| Feature | How it works |
|---|---|
| AI Quiz Generation | Send lecture title + description to OpenAI/Gemini → receive 5–10 multiple choice questions → save to DB |
| AI Lecture Overview | Send lecture title + description → receive 3–4 sentence summary → save to `Lecture.ai_summary` field |

Once generated, AI content is stored and reused — no repeated API calls for the same lecture.

## Data Models (Django)

### `UserProfile`
Extends Django's built-in `User` model.

| Field | Type |
|---|---|
| user | OneToOneField → User |
| total_score | IntegerField |
| avatar_url | URLField (optional) |

### `Lecture`
Represents a single lesson in the pathway.

| Field | Type |
|---|---|
| title | CharField |
| topic | CharField |
| youtube_url | URLField |
| description | TextField |
| ai_summary | TextField (cached, generated once) |
| order | IntegerField (defines pathway sequence) |
| created_at | DateTimeField |

### `Question`
A quiz question linked to a lecture.

| Field | Type |
|---|---|
| lecture | ForeignKey → Lecture |
| text | TextField |
| choices | JSONField (list of options) |
| correct_answer | CharField |

### `QuizAttempt`
Records a user's quiz attempt for a lecture.

| Field | Type |
|---|---|
| user | ForeignKey → User |
| lecture | ForeignKey → Lecture |
| score | IntegerField |
| total_questions | IntegerField |
| completed_at | DateTimeField |

## API Endpoints (Django REST Framework)

### Auth
| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/auth/register/` | Register new user |
| POST | `/api/auth/login/` | Login, returns JWT tokens |
| POST | `/api/auth/logout/` | Logout (blacklist refresh token) |

### Lectures
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/lectures/` | List all lectures (ordered pathway) |
| GET | `/api/lectures/:id/` | Get single lecture with AI summary |
| POST | `/api/lectures/` | Create lecture (admin only) |
| PUT | `/api/lectures/:id/` | Update lecture (admin only) |
| DELETE | `/api/lectures/:id/` | Delete lecture (admin only) |

### Questions
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/lectures/:id/questions/` | Get questions for a lecture (generates via AI if none exist) |

### Quiz
| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/quiz/submit/` | Submit answers, returns score, saves attempt |
| GET | `/api/quiz/attempts/` | Get current user's quiz history |

### Leaderboard
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/leaderboard/` | Get top users ranked by total score |

## Frontend Structure (Angular)

### Routes
| Path | Component | Description |
|---|---|---|
| `/login` | LoginComponent | Login and registration forms |
| `/lectures` | LectureListComponent | Lesson pathway overview |
| `/lectures/:id` | LectureDetailComponent | Video player + quiz |
| `/leaderboard` | LeaderboardComponent | Global ranking table |
| `/profile` | ProfileComponent | Personal stats and history |

### Services
- `AuthService` — handles login, logout, token storage
- `LectureService` — fetches lecture list and single lecture
- `QuizService` — fetches questions, submits answers
- `LeaderboardService` — fetches ranking data

### Angular Requirements Coverage
| Requirement | Implementation |
|---|---|
| 4+ click events | Submit quiz, logout, navigate to lecture, filter lectures |
| 4+ ngModel | Login form, register form, quiz answer selection, search/filter input |
| 3+ named routes | /login, /lectures, /lectures/:id, /leaderboard, /profile |
| HTTP interceptor | Attaches JWT Bearer token to every outgoing request |
| @for / @if | Loop over lectures, loop over questions, show score conditionally |
| Error handling | Display error messages on failed login, failed quiz submission, network errors |
